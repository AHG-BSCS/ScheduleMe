# Contributing Guidelines
Welcome to our open source project! We appreciate your interest in contributing to this project and making it better for everyone. To ensure a smooth and productive collaboration, please read and follow these guidelines when contributing to this project.

# How to contribute
1) Fork this repository to your GitHub account.
2) Clone your forked repository to your local machine.
3) Create a new branch to work on your changes: git checkout -b my-feature-branch
4) Make your changes and commit them with a descriptive commit message.
5) Push your changes to your forked repository: git push origin my-feature-branch
6) Submit a pull request to the main repository.

# Pull Request Guidelines
Before submitting a pull request, make sure you have done the following:

1) Rebased your branch on the latest main branch.
2) Tested your changes thoroughly to ensure they work as expected.
3) Added documentation for any new features or changes.
4) Followed the coding standards and conventions used in the project.

# Code of Conduct
This project adheres to a [code of conduct](https://github.com/AHG-BSCS1A/ScheduleMe/blob/master/CODE_OF_CONDUCT.md). By participating, you are expected to uphold this code. Please read the code of conduct before contributing.

Issue Tracker
If you find any issues or bugs in the project, [please create a new issue in the issue tracker](https://github.com/AHG-BSCS1A/ScheduleMe/issues). Make sure to provide as much information as possible about the issue, including steps to reproduce it and any error messages you receive.
